# Capture Knowledge Workflow

Turn implicit decisions into explicit documentation.

## Step 1: Identify Insight
- **Trigger**: "We just learned X about the system."
- **Context**: Is this architectural, process, or domain knowledge?

## Step 2: Select Home
- **Router**:
    - High-level? `docs/ai/design/`
    - How-to? `.agent/workflows/`
    - Edge case? `docs/ai/implementation/`

## Step 3: Write Doc
- **Action**: Create or append to the target file.
- **Format**: Use clear headers and code examples.

## Step 4: Link
- **Cross-Reference**: Update `README.md` or other docs to point to this new info.

## Step 5: Session Summary
- **Output**: Link to new knowledge artifact.
