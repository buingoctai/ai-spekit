# Requirements Phase

This directory stores the **Intent** of the Software Product Engineer. 

## Purpose
- Define **What** needs to be built and **Why**.
- Store Product Requirement Documents (PRDs) and Spec files.

## Workflow
1.  **Define Intent**: Create a `.md` or `.spec` file here describing the feature.
2.  **Review**: Ensure the problem statement and success criteria are clear.
3.  **Transition**: Once approved, move to the **Design** phase.
